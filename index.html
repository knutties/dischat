dischat

a javascript: bookmarklet

Turns any GitHub Discussions page into a Slack-style group chat — channels in the sidebar, messages in the middle, threaded replies in a side panel.

Install

Drag this button to your bookmarks bar. Then click it on any github.com/.../discussions page.

▸ dischat

How it works

  1. The bookmarklet fetches dischat.js from jsDelivr (a CDN mirror of this repo).
  2. The script scrapes the rendered DOM — author names, avatars, comment bodies, timestamps, and nested replies.
  3. It paints a full-screen overlay styled like Slack. The original GitHub page is untouched underneath.
  4. Click the bookmarklet again — or press Esc — to dismiss the overlay.

What you get

  • Single discussion view: top-level comments become messages in a channel; nested replies open in a side thread (Slack-style).
  • Discussions index view: each discussion is rendered as a channel in the sidebar and as a message preview in the list.
  • Day separators, grouped consecutive messages from the same author, OP badge on the original poster, "Answer" badge on accepted answers (when GitHub marks them).
  • The chat is read-only — clicking Reply on GitHub closes the overlay so you can use GitHub's normal comment box.

Manual install

If dragging doesn't work in your browser, right-click the button above and choose "Bookmark this link", or create a new bookmark with this URL:

javascript:(function(){var s=document.createElement('script');s.src='https://cdn.jsdelivr.net/gh/knutties/dischat@main/dischat.js?_='+Date.now();s.onerror=function(){alert('Dischat: failed to load.')};document.body.appendChild(s)})();
jsDelivr cache: changes to dischat.js propagate to the CDN within ~12 hours. To force a refresh, visit https://purge.jsdelivr.net/gh/knutties/dischat@main/dischat.js.

Caveats

  • GitHub's DOM changes — if the script can't find comments it shows a "no messages found" empty state. Selectors live in findCommentElements in dischat.js.
  • Read-only by design. Posting comments would require a GitHub OAuth token; out of scope for a bookmarklet.
  • Comment bodies are reused as-is from GitHub's already-rendered HTML, so they inherit GitHub's sanitization.
